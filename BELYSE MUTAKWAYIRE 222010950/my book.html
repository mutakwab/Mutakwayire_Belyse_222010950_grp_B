      2. Venue Table

        3. Venue Table database structure This table stores information about event venues,including venue ID,name,address,capacity,facilities available and any special requirements or restrictions. etc. It serves as a central repository for all event related data.


      3. User Table

        4. This table includes user id,username,password,fullname,email address,phone number,role level(e.g admin,event organizer, attendeeand),profile picture,registration date,last login,preference,event attended,event organized.this table allows the system to manage user authentication,track user activity and personalize user experience based on preferences and roles.


      4. Event Table

        5. This table stores information about each event such as event id,event name,description,data,time,location,and other relevant details.this table helps in preparation of the event because all needed details are provided.


      5. Photographers Table

        6. This table stores information about photographer id ,name,contact information such as email address and phone number,specialization,portfolio,availability,rate,experience,etc.this table would alloww event managers to keep track of photographers they have worked with.


      6. Orders Table

      Orders Table database structure This table Stores details about customers orders placed by the company or organization to customers or may customers can provide orders for what he/she wants to purchase, including order ID, customer ID, order date, total amount, order status etc.It helps in tracking and managing the procurement process.
